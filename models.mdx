FlexInference routes to **OpenAI**, **Google Gemini**, and **Anthropic**, and proxies **any** model for `default`, `priority`, and `auto` requests. The models below additionally support the **flex race** - the duration form of [`start_within`](/deadline-routing). Pass the model by its **alias** (for example `gpt-5.5` or `gemini-3.5-flash`), not a dated snapshot.

A **duration** `start_within` on a model that isn't on this list returns `400 model_not_flex_capable` - the flex race only works on flex-capable models. Those same models still work with `default`, `priority`, or `auto`, which proxy any model straight to its provider. For Gemini, `default` maps to Gemini's **standard** tier; Gemini has no `auto` tier, so `auto` on a Gemini model returns `400 auto_unsupported_for_gemini`.

## Anthropic (Claude)

Claude models route to Anthropic and are **proxy-only**: they support `default`, `priority`, and `auto`, but **not** the flex race. A duration `start_within` on a `claude-*` model returns `400 flex_unsupported_for_anthropic`, because Anthropic has no flex tier. `default` maps to Anthropic's **standard_only** service tier, `auto` lets Anthropic pick, and `priority` is best-effort (mapped to `auto`, since Anthropic rejects a literal priority tier). Every `claude-*` request **must** set `max_output_tokens` (`max_tokens` on `/v1/messages`); without it you get `400 missing_max_tokens`.

Claude is reachable through any caller format - `/v1/responses`, `/v1/chat/completions`, `/v1/interactions`, and the Anthropic-native [`/v1/messages`](/sdks#messages) - since FlexInference translates each to Anthropic's Messages API. The served tier is authoritative from Anthropic: a `standard_only` request is reported as `"service_tier": "standard"` on the response.

## Inputs

These models accept the input types you'd expect from OpenAI:

- **Text** — system, developer, user, and assistant messages.
- **Images** — pass image URLs or base64 data URLs. Data URLs are the most reliable.
- **Files** — PDFs and other file inputs supported by the Responses API.

Output is text (including tool calls, structured outputs, and reasoning). Streaming works on every model. On Gemini, `reasoning.effort` maps to a per-model `thinking_level` and may be clamped to the range that model supports.

<Note>
  Text, streaming, structured outputs, function calling, image input, file input, and web search work on **both** OpenAI and Gemini. For web search, send a Responses `web_search` tool; on Gemini we map it to `google_search`. Pass files as base64 data URLs (PDFs are the most reliable on Gemini).
</Note>

<Note>
  New flex-capable models are added as OpenAI ships them. If a model you expect is missing, check that you're using its alias and that OpenAI offers flex pricing for it.
</Note>
