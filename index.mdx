FlexInference is a drop-in router for the OpenAI API. You send the same Chat Completions or Responses requests you already send, with your own OpenAI key, and add one field — `start_within` — to tell us how long you're willing to wait. We route each request to the cheapest tier that meets your deadline, and fall back automatically if it can't.

<Info>
  FlexInference does not sell inference. You bring your own OpenAI key (BYOK), and OpenAI bills your account directly at whatever tier served the request. We add deadline-aware routing on top.
</Info>

## Why it exists

OpenAI offers the same models at different prices depending on how fast you need them. Flex (batch) rates are much cheaper than standard, but capacity isn't guaranteed. Most apps don't have the time to manage that trade-off per request.

With FlexInference you express the trade-off declaratively:

<CardGroup cols={3}>
  <Card title="Need it now" icon="bolt">
    `start_within: "priority"` routes to OpenAI's priority tier.
  </Card>
  <Card title="Can wait a bit" icon="hourglass-half">
    `start_within: "00h-00m-30s"` races flex against your deadline, then falls back to standard.
  </Card>
  <Card title="Standard" icon="equals">
    `start_within: "standard"` uses the default tier and pricing.
  </Card>
</CardGroup>

## Drop-in compatible

The base URL is the only thing you change. Everything else — streaming, tool calling, structured outputs, vision, reasoning — works exactly as it does against OpenAI.

```
https://api.flexinference.com/v1
```
